HTML

  • The head element contains information about the webpage
  • The body element represents the visible content shown to the user.
  • The !DOCTYPE tells the brower the document type, for example html.
  • The lang="en" tells the browser the page is in English.
  • charset="UTF-8" sets the character encoding. http-equiv sets and http header. viewport sets the default view.
  • The child elements are nested insit the parent element, such as the elements within the body tags.
  • The h1 tag is the first header. The order is important for assistive technology.
  • ul is bullet points, ol is a numbered list. li is a line item. p is a paragraph.
  • The img element gives info on images. src is an important attribute because it locates the image.
  • alt gives a description of the image. You also set the width and height here.
  • The br element creates a line of empty space. The a element stands for anchor, and it it creates a link. The url for the link is href.

CSS

  • inline CSS - style an element in the HTML file; quick and permanent; less flexible
  • uses "style="
  • internal style sheet contains CSS rules in the head section of the HTML; applies to all tagged page elements (ex, h1)
  • external CSS style sheet is most common way to style a webpage; allows us to keep CSS rules in a separate file, makes design changes easier
  • link file using in the HTML's after final element:
  • creates connection to external file; rel specifies the relationship between current and linked documents; href specifies the URL location of the external file
  • CSS Rule Structure: selector (ex: div); div {property: value; property: value;}; declaration = propert (ex: color): value (ex: blue); wild card symbol (*) applies the rule to all the elements visible on the page
  • to combine two selectors into a single CSS rule, use a comma (ex: header, footer) rather than repeating it (Do Not Repeat Yourself aka DRY code)
  • cd assets; touch style.css; git status; git checkout main; git checkout -b feature/add-css; VS CODE - open new style.css style sheet; add code in style.css
  • Add Style Using Class Attributes: designate a class attribute as the selector (class selector; ex: card); adds styles to certain elements, but not all
  • .card designates this as a class selector; review HTML to ensure the selector is assigned to the correct elements

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • Once you add your code, it is time to save to GitHub. Make sure you are on the correct branch.
  • Stage the changes: git add -A commits all files in the working branch, stages the changes.
  • Commit to the repository: git commit -m; -m associates a message with the commit.
  • Make sure current branch is up to date: git pull origin main
  • Push all changes: git push origin feature/add-css
  • Navigate to the repository on GitHub. Click the Pull Requests tabe and then New pull request button.
  • Make sure the base branch is main and comparison branch is your feature branch. Add a title and short description of the changes.
  • Click Create pull request, the Merge pull request, then confirm the merge request.
  • Close the GitHub issue in the Issues tab.
  • Navigate to main branch to update our local main branch: git checkout main
  • Pull the remote main branch to the local main branch: git pull origin main
