HTML

  • The head element contains information about the webpage
  • The body element represents the visible content shown to the user.
  • The !DOCTYPE tells the brower the document type, for example html.
  • The lang="en" tells the browser the page is in English.
  • charset="UTF-8" sets the character encoding. http-equiv sets and http header. viewport sets the default view.
  • The child elements are nested insit the parent element, such as the elements within the body tags.
  • The h1 tag is the first header. The order is important for assistive technology.
  • ul is bullet points, ol is a numbered list. li is a line item. p is a paragraph.
  • The img element gives info on images. src is an important attribute because it locates the image.
  • alt gives a description of the image. You also set the width and height here.
  • The br element creates a line of empty space. The a element stands for anchor, and it it creates a link. The url for the link is href.

CSS

  • inline CSS - style an element in the HTML file; quick and permanent; less flexible
  • uses "style="
  • internal style sheet contains CSS rules in the head section of the HTML; applies to all tagged page elements (ex, h1)
  • external CSS style sheet is most common way to style a webpage; allows us to keep CSS rules in a separate file, makes design changes easier
  • link file using in the HTML's after final element:
  • creates connection to external file; rel specifies the relationship between current and linked documents; href specifies the URL location of the external file
  • CSS Rule Structure: selector (ex: div); div {property: value; property: value;}; declaration = propert (ex: color): value (ex: blue); wild card symbol (*) applies the rule to all the elements visible on the page
  • to combine two selectors into a single CSS rule, use a comma (ex: header, footer) rather than repeating it (Do Not Repeat Yourself aka DRY code)
  • cd assets; touch style.css; git status; git checkout main; git checkout -b feature/add-css; VS CODE - open new style.css style sheet; add code in style.css
  • Add Style Using Class Attributes: designate a class attribute as the selector (class selector; ex: card); adds styles to certain elements, but not all
  • .card designates this as a class selector; review HTML to ensure the selector is assigned to the correct elements

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • Once you add your code, it is time to save to GitHub. Make sure you are on the correct branch.
  • Stage the changes: git add -A commits all files in the working branch, stages the changes.
  • Commit to the repository: git commit -m; -m associates a message with the commit.
  • Make sure current branch is up to date: git pull origin main
  • Push all changes: git push origin feature/add-css
  • Navigate to the repository on GitHub. Click the Pull Requests tabe and then New pull request button.
  • Make sure the base branch is main and comparison branch is your feature branch. Add a title and short description of the changes.
  • Click Create pull request, the Merge pull request, then confirm the merge request.
  • Close the GitHub issue in the Issues tab.
  • Navigate to main branch to update our local main branch: git checkout main
  • Pull the remote main branch to the local main branch: git pull origin main

JavaScript

  • Allows users to interact with websites - first add a variable to a JS file; second add conditional logic using JS; third incorporate arrays and for loops; fourth complete the Git flow
  • var = variable; var name = "string, of, topics" (all strings are surrounding by quotation marks); access by using the variable's name
  • console.log(var name) adds the console.log method to the file; the method can be used to output a message to the web console by adding an argument
  • Connect the script.js file to the index.html file (similar to how we linked the CSS style sheet style.css); add a link element inside the body element
  • Need to use Chrome DevTools to see it in action (right click in the browser, select Inspect); go to Console tab - here we see the value of the variable aka console.log method in action
  • Control Flow: the order in which a computer executes code in a script; order is very important
  • Using conditional statements and loops control the code's execution
  • If statements: code with only execute IF the statement meets a condition or is true
  • Falsy: 0, -0, On, "", undefined, null, or NaN; or Boolean value of false
  • Truthy: all other values, including Boolean value of true
  • Multiple conditions use else if. If the first condition is truthy, that code is executed. If the first condition is false, we move to the next else if option; if its truthy, it will be executed; if not, it will move on.
  • === is the strict equality operator; checks if two values are equal and returns a Boolean result (true if equal, false if unequal)
  • Have to manually change the variable value to test different values - solved w/ an Array
  • Arrays: store a group of various data within one variable; can perform a task on multiple pieces of data at once
  • var variable-name = ["data", "more data". "three datas"]
  • Arrays can hold multiple types of data as well (strings, numbers, etc)
  • Need additional information - the location of the data item in the array, aka an index
  • Arrays are zero-indexed and sequential - the first item is 0, not 1.
  • If we only wanted to access "more data", we would use variable-name[1];
  • Log it in the console to see it in action console.log(variable-name[1]);
  • for Loop uses the pattern of index to perform a task on all items in an array; much more efficient
  • A console.log for loop can tell the computer to go through each piece of data in an array and log them to the cosnole until there are no data items left
  • Need to know the index and the array's length
  • Array length will tell the for loop how many times the code will run; find the array using the length property
  • There are three important statements: 1) starting point of the loop; declare variable x and give it a value of 0 (var x = 0;); 2) the condition tells the loop how many times to run; as long as variable x is less than the length of the array, the loop keeps running; x < array.length; ; 3) final statement allows the array to iterate over each item; we know an index increases by 1 each time; x++ is short for x+1; now when the code loops, it will increase by 1
  • Commenting out code: makes it inactive without deleting, in case you need it later. Put /* before and */ after the block of code
  • A function is a set of instructions that tells the computer how to perform a task; different from a for loop because it doesn't automatically run
  • A function's name is followed by parentheses: function hellowWorld()
  • The syntax is similar to if statements and for loops, with the code inside { }
  • You have to call the function in order to run the code: type the name of the function into the console; add the function name() to the point in the code where you want the function to run.
